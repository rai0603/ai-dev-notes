AI 開發時代的真相

Vibe Coding 是什麼？

「會不會寫程式」這個問題，已經改變了。

❌ 叫 AI 寫程式（工具使用）
你是審核者，AI 是執行者
給 AI 一個任務，AI 執行，你審核結果。重點在「輸出的結果」。
太模糊。AI 不知道你的架構、你的限制條件，輸出的東西大概不是你想要的那種。
✅ Vibe Coding（協作方式）
你是方向決定者，AI 是加速器
你帶著想法和判斷力，跟 AI 一起把事情想清楚、做出來。重點在「過程的協作」。
我從來不是把需求貼給 Claude 然後等它輸出。我是在跟 Claude 對話的過程中，逐漸把一個模糊的想法變成可以執行的架構。
「前者是目的，後者是態度。Vibe Coding 不是一個工具的使用方式，它是一種跟 AI 協作的心態。」
— 陳建文

你真正需要的三件事

1
一台可以用的電腦
我個人是用 iMac 開發。但這裡有一個重要的提醒：如果你的 Mac 系統是 macOS 14 以下，有些工具（像是 Claude 的 Cowork 功能）無法使用。在開始之前先確認你的電腦環境。
2
耐心跟 AI 反覆溝通的心態
這是最容易被忽略的。Vibe Coding 不是一次下指令就完成，它是一個來回溝通、反覆調整的過程。如果你沒有耐心進行這個過程，你會在第一個 bug 出現的時候就想放棄。
3
好奇心和勇於發問的勇氣
你不需要知道所有答案，但你需要願意問問題。不知道某個名詞是什麼？問 AI。不確定某個決策對不對？問 AI。這個「問」的能力，比任何程式知識都更重要。

方法論

Claude Code 的正確使用方式

我把這一個月試錯出來的東西整理成可以複製的流程，讓你少走冤枉路。核心原則只有一個：在動手開發之前，先把事情想清楚。

1
先跟 Claude Chat 聊，不要直接開 Code
把你的想法用自然語言說清楚。不需要技術術語，就像跟朋友講話一樣。這個階段的目標是「把模糊的想法說清楚」，不是寫程式。
2
請 AI 搜尋有沒有類似的現成服務
讓 Claude 幫你搜尋是否已有相關的工具或競品，分析他們的架構和功能，整理成清單。你不需要重新發明輪子，你需要知道輪子已經長什麼樣子。
3
讓 AI 整理出完整的架構與規格書
請 Claude 根據你的想法和市場研究，產出一份規格書：功能清單、資料結構、API 需求、技術棧建議。這份規格書是後續所有工作的基礎。
4
問其他 AI：如果這個專案失敗，原因是什麼？
把規格書同時丟給 ChatGPT 和 Gemini，問他們：「如果這個想法失敗了，最可能的原因是什麼？怎麼解決？」用不同的 AI 做壓力測試，找出你可能沒想到的盲點。
5
整合後丟給 Claude Code，開始執行
把規格書、失敗分析、解決方案全部整合，丟給 Claude Code 確認工具和執行計畫，然後正式開始開發。從這一步開始，才是真正動手的時候。
6
針對每個功能逐一測試和調整
不要想一次做完所有功能再測試。每完成一個功能就測試一次，確認沒問題再繼續下一個。發現問題越早，修復成本越低。
「AI 很像許願池。你許什麼願，它就給你什麼。但就像電影裡的劇情，願望實現了，卻附帶了你不想要的東西。所以在許願之前，先想清楚怎麼正確地下指令、設定框架、說清楚你要避免什麼。」
— 陳建文的核心開發哲學
新手避坑

新手最常犯的三個錯誤

這三個錯誤我都犯過，現在整理出來讓你少走彎路。

❌ 壞的 PROMPT 方式
「幫我做一個登入功能。」
太模糊。AI 不知道你用什麼資料庫、想要什麼登入方式、有什麼限制條件。它會做出一個「某種」登入功能，但大概不是你想要的那種。
✅ 好的 PROMPT 方式
「我的專案用 Supabase 做資料庫。幫我實作 Email + Google OAuth 登入，登入後跳轉到 /dashboard 頁面，失敗時顯示錯誤訊息在表單下方，不要跳出彈窗。」
具體、有邊界、說清楚你要避免什麼。
📝
錯誤一：忘記記錄開發進度
Claude Code 的上下文是有限的。開發一段時間後，如果你沒有養成記錄進度的習慣，隔天回來時 AI 就忘了你要做什麼。你可能還要重新修改之前已經改過的地方，重工率很高。
解決方法：每次結束工作前，請 Claude 幫你整理一份「今天做了什麼、明天要繼續做什麼」的摘要，存成文字檔案。
✂️
錯誤二：同時修改太多東西
一次只解決一個問題。很多新手看到 AI 輸出了一大段程式碼，就想一次把所有功能都加進去。這樣做的結果是：出現問題時，你不知道是哪個部分出了錯。
解決方法：每次只給 AI 一個清楚的任務，做完測試，確認沒問題再繼續。
🔧
錯誤三：出錯時不知道怎麼處理
當 Claude Code 給你錯誤或跑偏的答案時，不要只是說「這樣不對，重新做一次」。這樣會讓 AI 做出另一個不對的版本。
正確做法：把錯誤訊息完整貼給 Claude，說明「這個結果跟我預期的不同，我預期的是＿＿，實際得到的是＿＿」，讓 AI 基於具體的差異來修正。
